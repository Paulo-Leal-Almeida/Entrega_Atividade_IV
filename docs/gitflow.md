# GitFlow e Versionamento Semântico

Branches recomendadas:
- `main` — produção
- `develop` — desenvolvimento
- `feature/*` — novas funcionalidades
- `release/*` — preparações de release
- `hotfix/*` — correções emergenciais

Commits semânticos (exemplos):
- `feat: adiciona validação de CPF`
- `fix: corrige bug no menu mobile`
- `docs: atualiza README`
- `style: ajusta cores e foco`
- `build: minifica assets`
